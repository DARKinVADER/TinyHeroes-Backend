🦸 TinyHeroes — Architecture

Tech stack: Angular 21 · ASP.NET Core 10 · EF Core 10 · PostgreSQL 16 · FluentStorage · Hugging Face AI · Docker (3 containers)

1. System Architecture Diagram

🔒 nginx routes all /api/* calls to the API container and serves the Angular SPA for everything else. In production, add TLS in front of nginx. The uploads_data volume is mounted into the API at /app/uploads and served via /api/images/url/{key} — no separate storage sidecar needed.

2. Object Storage — FluentStorage

Single interface, provider chosen by connection string — zero code changes to switch clouds

Environment Provider NuGet package STORAGE_CONNECTION_STRING
Local / dev Local disk
no extra container		 FluentStorage disk://path=/app/uploads
AWS Amazon S3 FluentStorage.AWS aws.s3://keyId=...;key=...;bucket=tinyheroes;region=eu-central-1
Azure Azure Blob Storage FluentStorage.Azure.Blobs azure.blob://account=...;key=...
GCP Google Cloud Storage FluentStorage.GCP.CloudStorage google.storage://bucket=tinyheroes;cred=<base64>
Registration in Program.cs 
// Register all provider modules at startup (only used when the connection string matches)
StorageFactory.Modules.UseAwsStorage();
StorageFactory.Modules.UseAzureBlobStorage();
StorageFactory.Modules.UseGoogleCloudStorage();

// Inject as singleton — connection string from config
builder.Services.AddSingleton<IBlobStorage>(_ =>
    StorageFactory.Blobs.FromConnectionString(
        builder.Configuration["Storage:ConnectionString"]!));
The IBlobStorage interface is injected into ImageService. No other layer knows which provider is active.
📦 Local development: docker-compose has 3 containers (postgres, api, frontend) — no storage sidecar. The uploads_data Docker volume is mounted at /app/uploads and images are served via the API. Switching to any cloud provider is a one-line change to STORAGE_CONNECTION_STRING in .env.

SOLID Principles — Backend

PrincipleApplication
S — Single ResponsibilityEach service handles one aggregate. Each repository encapsulates queries for one entity. Controllers do HTTP plumbing only.
O — Open/ClosedNew features = new services + repos + controllers. ExceptionHandlingMiddleware handles all typed exceptions via pattern matching.
L — Liskov SubstitutionAll repository implementations are interchangeable (EF Core today, Dapper tomorrow). IImageStorage → FluentImageStorage can be swapped.
I — Interface SegregationIImageStorage (store/get/delete) is separate from IAiImageService (generate). Repositories expose only what consumers need.
D — Dependency InversionDomain defines repository interfaces — Infrastructure implements. Application defines service interfaces — Infrastructure implements. Controllers depend on Application only.
🏗️ Key decisions: Repository per aggregate (not generic IRepository<T>). No AutoMapper — manual mapping in services. DTOs live in Application. One DependencyInjection.cs per project for clean DI registration.

Smart vs Dumb Component Pattern

TypeLocationResponsibilities
Smart (Pages)features/*/pages/Inject services · manage signal state · fetch data · pass data down via @Input · handle @Output events
Dumb (Components)features/*/components/Receive @Input() · emit @Output() · zero service injection · pure rendering
Feature Servicesfeatures/*/services/HTTP calls · signal-based state · feature-scoped (not providedIn: 'root')

SOLID Principles — Frontend

PrincipleApplication
S — Single ResponsibilityPages orchestrate. Components render. Services communicate with API + hold state. Never mixed.
O — Open/ClosedNew features = new lazy-loaded route. Shared components extended via composition, not modification.
L — Liskov SubstitutionAny component accepting a Child model works regardless of data source. Consistent model shapes.
I — Interface SegregationFeature services expose only what their feature needs. Shared components have minimal input contracts.
D — Dependency InversionPages depend on injected service abstractions. Components depend on @Input/@Output contracts only.
📡 Angular 21: standalone: true is the default — not written in decorators. Signals (signal(), computed()) for all reactive state. No RxJS Subjects for component state (reserved for HTTP streams). Lazy-loaded feature routes via loadChildren.

💡 JSON columns: WeekSummary.rankings stores [{childId, deeds, rank, prizeLabel}] · prizesAwarded stores {"1":"prize text","2":"...","3":"..."}. Snapshots are written once when the week/month closes — never mutated.

🔑 imageKey: GoodDeed.imageKey is a FluentStorage blob key (e.g. deeds/2024/abc123.png) — not a URL. Resolved to a served URL via GET /api/images/url/{key}.

🏠 familyId = null pattern is used consistently for both DeedPreset and PrizePreset: null = built-in entry visible to all families; familyId set = owned by that family. For PrizePreset specifically: scope + rank = null → reusable library entry; scope + rank set → active configured prize for that slot. No redundant isSystem flag — familyId is the discriminator.

👥 FamilyMember.relation is a free-text label set in My Profile (e.g. "Dad", "Mom", "Grandma") — distinct from role which is the permission level (Admin/CoParent).
